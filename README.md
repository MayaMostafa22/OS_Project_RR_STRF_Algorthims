# Round Robin vs SRTF Comparison Project

## Project Description
This project is a CPU scheduling simulator designed for the Operating Systems course. It implements and compares two scheduling algorithms: **Round Robin (RR)** and **Shortest Remaining Time First (SRTF)**. The main focus of the project is to evaluate fairness-oriented time-slicing versus efficiency-oriented shortest-remaining-time execution. The application features a desktop GUI built with Python and Tkinter that allows users to input process data (Arrival Time, Burst Time) and a Time Quantum, and visualizes the execution using separate Gantt charts. It also calculates and compares key metrics including Waiting Time (WT), Turnaround Time (TAT), and Response Time (RT).

## Requirements
* Python 3.x installed on your system.
* Tkinter (Usually included by default with standard Python installations).
* No external pip libraries are required.

## Build and Run Steps
1. Clone the repository to your local machine:
   `git clone <repository_url>`
2. Open your terminal or command prompt.
3. Navigate to the project directory:
   `cd <project_directory_name>`
4. Run the main application file (e.g., `main.py`) using Python:
   `python main.py` 
   *(Note: use `python3 main.py` depending on your OS configuration)*
5. Enter the processes data and the time quantum in the input panel, then execute to view the Gantt charts and comparison results.
